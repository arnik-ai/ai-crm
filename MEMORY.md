## ⚠️ قوانین طلایی کار (بسیار مهم — همیشه رعایت شوند)

1. **بدون اجازه و تأیید کاربر، هیچ کدی تغییر داده نشود و هیچ فایلی حذف نشود.**
2. **مراقب شکستن کدها باش** — هر تغییر باید قبل و بعدش با compile/test بررسی شود.
3. **کدها را کامل بخوان، حدس نزن.** قبل از ویرایش هر فایل، کلش را با دقت بخوان.
4. **همیشه دقت کن** — تغییرات کوچک و دقیق، نه بازنویسی‌های بزرگ بی‌دلیل.
5. **کدها ماژولار بمانند** — معماری Clean Architecture و جداسازی ماژول‌ها حفظ شود؛
   هر قابلیت در ماژول خودش، پشت اینترفیس، بدون وابستگی‌های درهم.
6. فایل `crm/.env` (ریشه) مالِ پروژه‌ی دیگری (rankiiup) است — **دست نزن، حذف نکن، منتقل نکن.**
7. **همیشه کدها را داکیومنت کن (مستندسازی).** هر کدی که می‌نویسی یا تغییر می‌دهی باید
   توضیح داشته باشد تا بعداً قابل‌فهم باشد. این یعنی:
   - **داخل کد:** docstring فارسی برای هر کلاس/تابع + کامنت برای منطق غیربدیهی
     (چرا این‌طور، نه فقط چه‌کار). الگوی موجود همین است — حفظش کن.
   - **در MEMORY.md:** بعد از هر کارِ مهم، یک مورد به «کارهای انجام‌شده» + تاریخچه‌ی
     commit اضافه کن (چه شد، چرا، کدام فایل‌ها).
   - **کارهای قبلی هم:** اگر کدی بدون توضیح ماند، هنگام عبور از آن، مستندش کن.
8. **همه‌ی تاریخ‌های نمایشی همیشه شمسی باشند.** داده‌ی تاریخ به‌صورت میلادی/UTC
   ذخیره می‌شود (برای محاسبه‌ی دقیق)، ولی در نمایش **همیشه** شمسی شود:
   - فرانت: از `faDate` (بدون ساعت) یا `faDateTime` (با ساعت) استفاده کن — هرگز تاریخِ
     خامِ ISO/میلادی را مستقیم نشان نده. (تقویم شمسی داخلِ مرورگر است: `Intl` با `fa-IR`.)
   - بک‌اند: برای برچسبِ شمسی از `src/shared/utils/jalali.py` استفاده کن.
   - **ورودی تاریخ:** هرگز `<input type="date">` (تقویمِ میلادیِ مرورگر) استفاده نکن؛
     از `components/JalaliDatePicker` استفاده کن (تبدیل در `lib/jalali.ts`، خروجی ISO میلادی).
9. **همیشه RTL و فارسی را رعایت کن.** کلِ UI راست‌چین فارسی است (`dir="rtl"` در layout).
   فقط داده‌های ذاتاً لاتین (شماره موبایل، مبلغ، کد رهگیری) می‌توانند `dir="ltr"` باشند؛
   تاریخ/متنِ فارسی هرگز نباید `ltr` شود. اعداد در نمایش با `faNum`/`faDigits` فارسی شوند.

## 📦 معماری پروژه (ماژولار)

```
backend/  → FastAPI، Clean Architecture، ۶ ماژول مستقل:
            identity (کاربر/نقش/JWT/RBAC)
            crm (دانشجو/دوره/تگ/یادداشت/پیگیری/مراحل فروش)
            telephony (تماس/ضبط/وب‌هوک + Provider قابل‌تعویض)
            ai_analysis (LangGraph + Whisper + GPT-5.5 + امتیازدهی)
            analytics (داشبورد)
            assistant (دستیار چت)
frontend/ → Next.js 15، RTL فارسی، داشبورد گرافیکی
docs/     → ۱۲ سند معماری فارسی
```

هر ماژول لایه‌بندی شده: `domain/` → `application/` (+ports) → `infrastructure/` → `api/`.
**قانون:** دامنه به فریم‌ورک وابسته نیست؛ Providerها پشت اینترفیس (Open/Closed).

> 📖 **مرجع کلیدی:** [`docs/11-CODE-GUIDE.md`](docs/11-CODE-GUIDE.md) — راهنمای مرور کدِ
> فایل‌به‌فایل (هر فایلِ بک‌اند/فرانت چه‌کار می‌کند + توابع کلیدی). قبل از کار روی هر
> فایل، اول این را ببین. **هر فایل/تابع جدید را همان‌جا ثبت کن** (قانون طلایی #۷).
