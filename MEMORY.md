## ⚠️ قوانین طلایی کار (بسیار مهم — همیشه رعایت شوند)

1. **بدون اجازه و تأیید کاربر، هیچ کدی تغییر داده نشود و هیچ فایلی حذف نشود.**
2. **مراقب شکستن کدها باش** — هر تغییر باید قبل و بعدش با compile/test بررسی شود.
3. **کدها را کامل بخوان، حدس نزن.** قبل از ویرایش هر فایل، کلش را با دقت بخوان.
4. **همیشه دقت کن** — تغییرات کوچک و دقیق، نه بازنویسی‌های بزرگ بی‌دلیل.
5. **کدها ماژولار بمانند** — معماری Clean Architecture و جداسازی ماژول‌ها حفظ شود؛
   هر قابلیت در ماژول خودش، پشت اینترفیس، بدون وابستگی‌های درهم.
6. فایل `crm/.env` (ریشه) مالِ پروژه‌ی دیگری (rankiiup) است — **دست نزن، حذف نکن، منتقل نکن.**
7. **همیشه کدها را داکیومنت کن (مستندسازی).** هر کدی که می‌نویسی یا تغییر می‌دهی باید
   توضیح داشته باشد تا بعداً قابل‌فهم باشد. این یعنی:
   - **داخل کد:** docstring فارسی برای هر کلاس/تابع + کامنت برای منطق غیربدیهی
     (چرا این‌طور، نه فقط چه‌کار). الگوی موجود همین است — حفظش کن.
   - **در MEMORY.md:** بعد از هر کارِ مهم، یک مورد به «کارهای انجام‌شده» + تاریخچه‌ی
     commit اضافه کن (چه شد، چرا، کدام فایل‌ها).
   - **کارهای قبلی هم:** اگر کدی بدون توضیح ماند، هنگام عبور از آن، مستندش کن.

## 📦 معماری پروژه (ماژولار)

```
backend/  → FastAPI، Clean Architecture، ۶ ماژول مستقل:
            identity (کاربر/نقش/JWT/RBAC)
            crm (دانشجو/دوره/تگ/یادداشت/پیگیری/مراحل فروش)
            telephony (تماس/ضبط/وب‌هوک + Provider قابل‌تعویض)
            ai_analysis (LangGraph + Whisper + GPT-5.5 + امتیازدهی)
            analytics (داشبورد)
            assistant (دستیار چت)
frontend/ → Next.js 15، RTL فارسی، داشبورد گرافیکی
docs/     → ۱۱ سند معماری فارسی
```

هر ماژول لایه‌بندی شده: `domain/` → `application/` (+ports) → `infrastructure/` → `api/`.
**قانون:** دامنه به فریم‌ورک وابسته نیست؛ Providerها پشت اینترفیس (Open/Closed).

## ✅ کارهای انجام‌شده (به ترتیب)

1. **معماری کامل + اسکلت** — ۹ سند فارسی، ساختار ماژولار، ERD، LangGraph.
2. **پیاده‌سازی واقعی** — سرویس داشبورد، تماس‌ها، CRM (courses/tags/stages)،
   ذخیره transcript، Audit log، Rate limiting، seed، ۱۸ تست.
3. **migration اولیه** — ساخت خودکار جدول‌ها از `sql/schema.sql` + seed در فاز release پارس‌پک.
4. **اتصال سیموتل** — `SimotelProvider` (رویدادهای IncomingCall/OutgoingCall/CDRQueue)،
   endpoint `/webhooks/simotel`، ۱۰ تست. بدون تغییر دامنه (لطف معماری ماژولار).
5. **داشبورد گرافیکی زیبا** — StatCard رنگی، نمودار Area روند تماس، دونات سرنخ،
   قیف فروش، فونت Vazirmatn، Sidebar گرادیانتی. endpoint `calls-trend`.
6. **حالت دمو فرانت** — `frontend/src/lib/demoData.ts` + سوییچ در `api.ts`
   (`NEXT_PUBLIC_DEMO=1`). داده‌ی فیک واقعی (رشته‌ی کنکوری تجربی/ریاضی/انسانی،
   پایه، هدف دانشگاهی) برای دیدن ظاهر بدون دیتابیس. هر وقت دیتابیس آمد، خاموش می‌شود.
7. **شماره موبایل فقط از سیموتل** — تأیید شد که موبایل از `event.caller_number` می‌آید
   نه از AI؛ مدل `ExtractedInfo` عمداً فیلد mobile ندارد. کامنت محافظ گذاشته شد.
8. **نشان اطمینان AI** — فیلد `confidence` در `ExtractedInfo` + نشان رنگی در صفحه‌ی
   تماس‌ها (سبز/زرد/قرمز=نیاز به بازبینی) + دکمه‌ی ویرایش (ظاهر؛ منطقش با دیتابیس وصل می‌شود).
9. **زیباسازی + یوزرفرندلی** — صفحه‌ی دانشجویان و تماس‌ها: جستجو، فیلتر، آواتار،
   مرتب‌سازی بر اساس امتیاز، کارت‌های آماری، حالت خالی، ته‌رنگ پاستیلی، راهنمای امتیاز سرنخ.
10. **تم تیره سرمه‌ای** — پس‌زمینه `#334766` (در `globals.css`)، سرتیترها سفید.
11. **ریسپانسیو کامل (موبایل/دسکتاپ، اندروید/iOS)** — viewport در layout،
    Sidebar با منوی همبرگری روی موبایل، چیدمان `flex-col md:flex-row`،
    پدینگ `p-4 md:p-8`، جدول‌ها `overflow-x-auto` + `min-w`.
12. **دکمه‌ی برگشت RTL** — `BackButton.tsx` (فلش راست، فارسی) در همه‌ی صفحه‌ها.
13. **PWA** — قابل نصب روی اندروید/iOS/دسکتاپ (`ServiceWorkerRegister.tsx`).
14. **سخت‌سازی پروداکشن** — Health Check واقعی (`/healthz`/`/readyz`)، کش داشبورد،
    rate limit عمومی + gzip + statement_timeout.
15. **ورود با پیامک (OTP)** — جریان امن: کد ۵رقمی، هش HMAC در Redis (TTL=۱۲۰ث)،
    مقایسه‌ی time-constant، سقف ۵ تلاش، cooldown=۶۰ث + Auth Guard صفحات.
16. **پنل مدیریت کاربران** — افزودن/ویرایش/فعال‌سازی اعضای تیم (RBAC؛ کاربرِ فقط-OTP
    با ایمیل داخلی `{mobile}@otp.local`).
17. **ثبت فیش فروش واقعی** — جدول `sales` + لیست محصولات + مدت برنامه؛ محصول «برنامه»
    الزاماً `program_months` دارد (validator) + پیگیری خودکار +۵ روز پس از فروش.
18. **صفحه‌ی اصلی «کارهای روز»** — هاب روزانه‌ی نیرو (پیگیری‌های امروز، تماس‌های بدون
    اقدام، ازدست‌رفته، یادآور تمدید) به‌عنوان صفحه‌ی ورودی (`/tasks`).
19. **معدل + گزارش‌ها + پیام‌ها (commit `0b77153`)** —

20. **تطبیق با چک‌لیست کارفرما (commit `e5715f2`)** — سه مورد جامانده رفع شد:
    - 🐞 **باگ ثبت ناقص دانشجو:** `create()` فیلدهای شهر/رشته/پایه/هدف/معدل را دور می‌ریخت

    - 📞 **تعداد تماس هر فرد:** subquery شمارش تماس در `StudentService.list` (کنار نام).
    - 🔁 **مشتریان چندبارخرید:** `GET /sales/repeat-customers` (تعداد خرید + تاریخ هر خرید
      + فاصله‌ی روز بین خریدها + جمع کل) + بخش جدید در صفحه‌ی گزارش‌ها + داده‌ی دمو.
    - ✅ تأییدشده: دسته‌بندی (Grade×field)، وضعیت تماس (۵ حالت)، منبع تماس (۷ گزینه) دقیقاً
